💡 使用说明 & 关于

📝 使用说明

📍 本地接口（核心用法）

将上方"本地接口信息"里的 Base URL 和 API Key 填入任何支持 OpenAI 格式的工具（ChatBox、NextChat、SillyTavern、Dify 等）即可使用。网关自动在多个上游提供商间轮询，单渠道挂了无感切换。

📊 实时监控

展示所有上游模型健康状态、SLA 可用率、平均/极速/最慢延迟、探针次数。模型名旁的状态点代表上次轮询结果：🟢 绿色=正常 / 🔴 红色=异常（限流或报错）/ ⚪ 灰色=尚未巡检。后台每 5 分钟自动巡检，也可点右上角"立即检测"手动触发。

📏 上下文长度

直接点击监控表格"上下文窗口"列的数字即可修改，回车保存。

🔀 路由配置

创建自定义路由组并勾选成员模型，客户端 model 字段填路由组名即命中（如 1m、256k），网关在组内按可用率/延迟加权择优。

⚙️ 模型管理

每个提供商卡片支持：编辑连接信息、勾选启用的模型、删除。添加提供商时自动拉取上游可用模型列表。

📦 管理输出模型

查看本网关对外暴露的所有模型 ID（格式 提供商名-模型名），可一键复制。

📈 消耗统计

按近24小时 / 7 天 / 30 天统计请求数、输入/输出/合计 Token，按"渠道·模型"维度明细展示。

🛡️ 智能容灾

熔断机制：某模型连续失败 3 次熔断 60 秒，恢复后自动重置。路由组按质量分+延迟加权排序候选。流式中断时自动切换下一个候选模型继续输出，用户无感。
